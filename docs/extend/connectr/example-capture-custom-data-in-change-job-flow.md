## Scenario
An HR team wants to capture custom approval information before a *job change* is submitted.

1. An Extend Developer builds an edit task using the Staffing Event business object as context.
2. The task does not include a blocking validation, making it optional or informational.
3. A Workday Admin inserts this task as the second step in the Change Job flow.

### Next Steps

Users view the custom step during the Change Job process but can skip it if needed.
