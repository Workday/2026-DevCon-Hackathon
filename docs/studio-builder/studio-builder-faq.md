* **What is Studio Builder?**: Studio Builder is a Workday Developer Site tool that enables you to maintain existing Studio integrations with complex requirements, including those that rely on custom code. Unlike Workday Studio, its full-featured Eclipse-based companion, Studio Builder is lightweight and, as one of the tools on the Workday Developer Site, is entirely browser-based.<br><br>
* **How do I access Studio Builder?**: Studio Builder is available on the Workday Developer Site at https://developer.workday.com/.<br><br>
* **Do I need to install anything?**: Studio Builder is entirely browser-based. No downloads or additional installations are required.<br><br>
* **Will my existing integrations work on Studio Builder?**: Integrations built with the desktop Workday Studio target Workday's legacy runtime, known as IR1. Studio Builder requires the newer, much-improved IR2. Integrations built with the desktop Workday Studio must be migrated to IR2 before you can edit them in Studio Builder. Migration happens automatically the first time you deploy a IR1 integration in Studio Builder.<br><br>
* **Is Java supported on Studio Builder?**: Studio Builder supports Java 17 but doesn't have a Java compiler. Use your normal IDE for Java development.<br><br>
* **How do I import an assembly to Studio Builder?**: From a `.clar` file, a project folder `.zip`, or directly from a tenant.<br><br>
* **What are Studio Builder's Limitations?**: Studio Builder doesn't have native Java development functionality and has no AI Copilot.<br><br>
* **Is Studio Builder production-ready?**: Studio Builder is still in development. We don't recommend its use in production. To create production-ready integrations, consider creating an integration app with Orchestration Builder.
