> **Note:** The version of Studio Builder available at Workday DevCon 2026 is a work in progress and not fully representative of the features expected in the finished product.

Studio Builder is a Workday Developer Site tool that enables you to maintain existing Studio integrations with complex requirements, including those that rely on custom code. Unlike Workday Studio, its full-featured Eclipse-based predecessor, Studio Builder is lightweight and, as one of the tools on the Workday Developer Site, is entirely browser-based.

Studio Builder runs on the latest Workday integration runtime, known as Integration Runtime 2 (IR2).

Its benefits can be summarized under these headings:

* **Developer Site Experience**
  * Because Studio Builder is part of the Workday Developer Site, you can manage your Studio integrations and custom apps behind a single login.
  * Developer Site documentation is expansive and regularly updated.
* **Utility**
  * Studio Builder provides a greatly simplified experience, enabling you to quickly and easily edit existing assemblies in-browser without having to download and configure an IDE.
  * Upgrades, fixes, and new capabilities can be rolled out more regularly with no user effort required.
  * Studio Builder supports the migration of assemblies based on the original Integration Runtime (IR1) to the newer, much-improved IR2.
* **Security**
  * Studio Builder enables you to centralize developer tenant access, security, and governance on the Workday Developer Site.
  * Studio Builder developers aren't exposed to potential security vulnerabilities in an external IDE.
* **Integration Runtime 2**
  * IR2 offers an elevated, FedRAMP-compliant level of security.
  * IR2 provides seamless interoperability with new and existing services such as App Hub.
  * Because IR2 provides classpath separation, you can package only the required library versions.
  * IR2 is up to 40% faster than IR1.
  * The majority of existing IR1 custom integrations can be dynamically migrated to IR2 with no customer input required.
  * In the small number of cases where manual intervention is required, Studio developers will be provided with necessary tools and documentation to guide them through the process.
