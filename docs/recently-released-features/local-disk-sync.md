# Product Update: Local Disk Sync - Now Generally Available

**Local Disk Sync** is now Generally Available (GA) for all Extend customers!

## What’s New

Enjoy seamless, automatic syncing between the builder tools in the browser and your local file system- enabling smoother integration with your preferred version control systems and CI/CD workflows while accelerating your development cycles.

## Key Capabilities
* You can now open your locally stored Extend and Integration apps directly within App Builder and Orchestration Builder
* Changes made in the builder tools in the browser are automatically reflected on your local disk and vice-versa

## Impact
* Eliminate repeated manual file downloading and uploading to integrate third party version control systems and development tools
* Reduce friction implementing CI/CD pipelines
* Accelerate your development cycles - focus on building your apps rather than file management

## How to Access
* To get started, create a new app or identify an existing one you'd like to edit with Local Disk Sync, then download the app zip to your local file system — this will serve as your local repository for the app.
* To activate Local Disk Sync, navigate to the Developer Site home page and click either “Build an Extend App” or “Build an Integration App”. From there, select Select the “Open a Local Folder” and provide the local folder that you previously downloaded.

## Considerations & Limits
* You must use a Chromium-based browser to use Local Disk Sync
* The app saved locally must first be created in App Hub for the same Developer Site organization you are accessing App Builder or Orchestration Builder in.
* You must press “Open a Local Folder” to re-associate your folder every time you want to open your app in a local folder session in App Builder or Orchestration Builder.
